# Архитектура админ-модуля

## Цель
Создать модуль администрирования каталога товаров с доступом по адресу `/admin` с паролем.

## Техническое решение

### 1. База данных (SQLite)
- **Почему SQLite**: Простая, встроенная, не требует отдельного сервера, легко контейнеризируется
- **Структура таблиц**:
  - `categories` - категории товаров
  - `products` - товары
  - `users` - пользователи админ-панели (для аутентификации)

### 2. API слой
- Создать API для работы с БД
- Эндпоинты:
  - `GET /api/categories` - получить все категории
  - `POST /api/categories` - создать категорию
  - `PUT /api/categories/:id` - обновить категорию
  - `DELETE /api/categories/:id` - удалить категорию
  - `GET /api/products` - получить все товары
  - `POST /api/products` - создать товар
  - `PUT /api/products/:id` - обновить товар
  - `DELETE /api/products/:id` - удалить товар
  - `POST /api/login` - аутентификация

### 3. Админ-интерфейс
- Отдельный роут `/admin`
- Компоненты:
  - `AdminLoginView.vue` - форма входа
  - `AdminDashboardView.vue` - главная панель
  - `AdminCategoriesView.vue` - управление категориями
  - `AdminProductsView.vue` - управление товарами
  - `AdminProductForm.vue` - форма создания/редактирования товара

### 4. Контейнеризация
- Docker контейнер для SQLite
- Монтирование локальной папки `data` в контейнер
- Использование официального образа `sqlite:3`

### 5. Интеграция с существующим проектом
- Замена статических данных в `src/lib/data.ts` на данные из БД
- Создание API middleware для обработки запросов
- Использование существующих компонентов UI

## Структура проекта

```
src/
├── api/
│   └── admin.ts          # API клиент для админ-панели
├── components/
│   └── admin/
│       ├── AdminLogin.vue
│       ├── AdminDashboard.vue
│       ├── AdminCategories.vue
│       ├── AdminProducts.vue
│       └── AdminProductForm.vue
├── views/
│   └── AdminView.vue     # Главный компонент админ-панели
├── stores/
│   └── useAdminStore.ts  # Pinia store для админ-данных
└── lib/
    └── db.ts             # Работа с БД

docker/
└── admin/
    ├── Dockerfile        # Контейнер для админ-сервера
    └── docker-compose.yml # Настройка контейнеров
```
